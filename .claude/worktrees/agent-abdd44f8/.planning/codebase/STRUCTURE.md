## Directory Purposes

**src/**
- Purpose: All TypeScript source code
- Contains: CLI orchestration, API clients, type definitions
- Key files: `index.ts` (entry), `types.ts` (schemas), `costco.ts` (fetch), `actual.ts` (import)

**.planning/codebase/**
- Purpose: Generated architecture documentation
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Key files: This file and sibling analysis documents

## Key File Locations

**Entry Points:**
- `src/index.ts`: CLI orchestration, argument parsing, main execution flow

**Configuration:**
- `tsconfig.json`: TypeScript compiler options (ES2022 target, bundler resolution)
- `package.json`: Dependencies and scripts (only `npm start` defined)
- `.env`: Environment variables for Costco and Actual Budget auth (gitignored)

**Core Logic:**
- `src/costco.ts`: Costco GraphQL API client with pagination and detail fetching
- `src/actual.ts`: Actual Budget SDK wrapper with transaction transformation
- `src/types.ts`: All TypeScript interfaces for API responses and data models

**Runtime Artifacts:**
- `costco-receipts.json`: Backup of fetched receipt data (generated, not committed)
- `dist/`: Compiled JavaScript output (generated, not committed)

## Naming Conventions

**Files:**
- `.ts` extension for all TypeScript source files
- kebab-case for directory names (none currently, flat structure)
- camelCase for exported function names
- PascalCase for type/interface names

**Functions:**
- Verb-first naming: `fetch*`, `import*`, `connect*`, `shutdown*`
- Examples: `fetchAllReceipts()`, `importReceipts()`, `connectActual()`, `parseArgs()`

**Types:**
- PascalCase with descriptive suffix: `*Detail`, `*Summary`, `*Response`
- Examples: `ReceiptDetail`, `ReceiptSummary`, `ReceiptListResponse`, `ItemDetail`

**Variables:**
- camelCase for all locals and module-level variables
- UPPER_SNAKE_CASE for constants (e.g., `GRAPHQL_URL`, `DATA_FILE`, `LIST_QUERY`)

**Exports:**
- Named exports for all functions: `export async function functionName()`
- Named exports for all types: `export interface InterfaceName {}`
- All modules use ES modules (import/export syntax)

## Where to Add New Code

**New Feature (e.g., support for returns/refunds):**
- Primary code: `src/index.ts` (add CLI flag + logic)
- Types: `src/types.ts` (add interfaces if needed)
- API client: `src/costco.ts` or `src/actual.ts` depending on which system

**New API Integration (e.g., export to different budgeting tool):**
- Create new file: `src/<tool-name>.ts` (follow pattern of `actual.ts`)
- Export async functions: `connect<Tool>()`, `import<Type>()`, `shutdown<Tool>()`
- Add CLI flag in `src/index.ts` to select which tool to use
- Add types to `src/types.ts` if needed

**New Transformation Logic (e.g., category mapping):**
- Utility function: Add to `src/actual.ts` or new `src/transforms.ts`
- Keep transformation logic close to where it's consumed
- Avoid creating new module unless shared by multiple layers

**Testing (if adding test files):**
- Location: `src/__tests__/` (co-locate with source by feature)
- Naming: `<module>.test.ts` or `<module>.spec.ts`
- No test infrastructure currently present; see TESTING.md

## File Modification Guide

**To modify CLI behavior:**
- Edit: `src/index.ts` parseArgs() and main()
- Don't edit: API layers or types unless interface changes

**To add a new Costco API field:**
- Edit: `src/types.ts` (add to ReceiptDetail or nested interface)
- Edit: `src/costco.ts` DETAIL_QUERY (add field to GraphQL selection)
- Edit: `src/actual.ts` if field affects transaction transformation

**To change transaction format for Actual Budget:**
- Edit: `src/actual.ts` importReceipts() function
- Reference: `ReceiptDetail` structure in `src/types.ts`
- Test: Run with `--import-only` to avoid refetching

**To adjust authentication:**
- Edit: `src/costco.ts` buildHeaders() (Costco)
- Edit: `src/actual.ts` connectActual() (Actual Budget)
- Update: `.env.example` to document new variables
